1970年Rudolf Bayer提出的B+树，灵感源自平衡二叉树和 B 树，为外存设计，减少IO次数

  1. 索引页数量少
  2. 再平衡时，B+ 树算法调整的索引节点数量也很少
  3. 索引容量足够大:3 层索引可承载 2000 万行数据，4 层索引可承载 200 多亿行
  4. 索引页少，可以将所有索引全部载入内存，读取索引的磁盘 I/O 趋近于 0
